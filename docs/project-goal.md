# Project Goal

## One-Sentence Summary

利用闲置手机、电脑和平板构建一个无主节点的家庭存储池，让用户像使用网盘一样使用统一存储空间，而无需关心文件实际位于哪台设备上。

## Background

闲置设备存在大量未利用的存储资源。

这些资源：

- 分散
- 不统一
- 不方便访问

用户往往需要额外购买 NAS、硬盘、小主机才能获得统一存储体验。

PocketCluster 希望直接利用已有设备形成统一存储空间。

## Target User

具备一定折腾能力的个人用户和极客用户。

拥有：

- 旧手机
- 旧平板
- 旧电脑
- 闲置存储空间

希望将其重新利用。

## Core Problem

当前闲置设备存在大量未利用的存储资源，这些资源分散、不统一、不方便访问。

## Core Value

用户获得：

```
统一存储空间
```

而不是：

```
多个设备
```

## Project Positioning

PocketCluster 的定位是：

```
闲置设备资源池
```

第一能力是：

```
存储
```

未来理论上还可以扩展：

- 计算
- GPU
- AI推理
- 媒体转码

但 MVP 完全不考虑这些。

## First-Version Goal

实现 Windows、Mac、Android 三节点资源池。

上传文件后：

- 自动切 Chunk
- 自动生成副本
- 任意节点可访问
- 节点离线后仍可读取
- 节点恢复后自动同步

## Non-Goals

MVP 不包括：

- 计算资源池
- GPU 调度
- AI 推理
- 媒体转码
- 公网节点
- 纠删码
- 权限系统
- SMB / WebDAV

## Success Criteria

- 三台设备可形成统一存储空间
- 上传文件后自动切 Chunk 并生成副本
- 任意节点可访问任意文件
- 单节点离线后文件仍可读取
- 节点恢复后自动同步
- 用户体验接近网盘
